# Phase 1.5 を "good enough" 宣言し Phase 2 に進む

## Status

Accepted

## Context

Phase 1.5（ハーネス自身のコード品質改善）は 24 項目中 14 完了、10 未着手の状態。
残り 10 項目（CLI-01 pr.rs 移動、ERR-09b activate.rs 分割、RVW-01/02 パーサー統合、構造的ロック等）は全て「sotp のコードを綺麗にする」リファクタリングであり、Phase 2（仕様品質）や Phase 3（テスト生成パイプライン = Moat）をブロックしない。

プロジェクト開始から Phase 1.5 まで、全ての投資がインフラ整備に向いており、Moat に 1 行もコードを書いていない。

## Decision

Phase 1.5 を "good enough" とし、残り 10 項目を延期して Phase 2 に進む。

INF-19（domain-purity CI, PR #53）と INF-20（conch-parser infra 移動, PR #54）を最後のタスクとして完了させ、Phase 1.5 を切り上げる。

## Rejected Alternatives

- **Phase 1.5 を完了してから Phase 2**: 残り 10 項目は推定 2.5 日。Moat 到達がその分遅れる。リファクタリングは Phase 3 後でも可能
- **Phase 1.5 と Phase 2 を並行**: 文脈切替コストが高い。1 人開発では直列の方が効率的

## Consequences

- Good: Moat 到達が ~2.5 日早まる
- Good: 「完璧にしてから次へ」パターンを断ち切れる
- Bad: pr.rs (1857行), activate.rs (1826行) の肥大化が残る
- Bad: pub(crate) による構造的ロックが未実施（内部 API が不安定なまま）

## Reassess When

- Phase 3 完了後に残りタスクの必要性を再評価
- pr.rs / activate.rs の変更頻度が高く、肥大化が開発速度に影響する場合
